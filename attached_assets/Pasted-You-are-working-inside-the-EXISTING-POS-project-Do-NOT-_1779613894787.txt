E1.1 — Query
Add to src/db/queries/operations.ts (or a new ledger.ts):

  getLedgerForPeriod(fromDate, toDate) →
    SELECT
      le.id,
      le.entry_date,
      le.created_at,
      a.name AS account_name,
      a.type AS account_type,
      le.type AS direction,           -- 'credit' or 'debit'
      le.amount,
      le.ref_type,
      le.ref_id,
      le.description
    FROM ledger_entries le
    LEFT JOIN accounts a ON le.account_id = a.id
    WHERE le.entry_date BETWEEN ? AND ?
    ORDER BY le.entry_date ASC, le.created_at ASC

E2.1 — Query
Add to src/db/queries/reports.ts (or use the existing getReport function
if it already covers most of this — extend rather than duplicate).

  getProfitAndLoss(fromDate, toDate) → {
    sales_gross,                 // SUM(total_amount) WHERE status IN ('active','partially_returned')
    returns_total,               // SUM(total_amount) WHERE status = 'returned'
    sales_net,                   // sales_gross - 0 (returned invoices excluded above)
    cogs,                        // SUM(unit_cost * quantity) for active+partial invoices
    gross_profit,                // sales_net - cogs
    expenses_total,              // SUM(amount) FROM expenses
    expenses_by_category,        // [{ category_name, total }]
    topup_profit,                // SUM(profit) FROM topups
    maintenance_revenue,         // SUM(final_amount) FROM maintenance_jobs WHERE status='delivered'
    other_income,                // topup_profit + maintenance_revenue
    net_profit,                  // gross_profit + other_income - expenses_total
    period: { fromDate, toDate }
  }

All status filters must match the conventions established in Phase 1.
