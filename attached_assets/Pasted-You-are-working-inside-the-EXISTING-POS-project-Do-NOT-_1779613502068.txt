D2.1 — getOpenDayPreview(targetDate) → returns the same shape as

commits. Reuse getDailySummary from operations.ts where possible.

D2.2 — closeDay(targetDate, cashCounts: { accountId, actualCash }[], notes)
- Validate: target date is today or earlier (no future closes).
- Validate: no existing row in day_closures for this date.
- For each cash-type account in cashCounts:
  - Get current `accounts.balance`.
  - diff = actualCash - balance
  - If diff != 0: append a ledger_entries row with ref_type='eod_reconciliation',
    type 'credit' if diff>0 else 'debit', amount=Math.abs(diff), description
    "تسوية إقفال يومي: {account_name} — الفرق {diff}".
  - Update accounts.balance = actualCash.
- Compute the closing snapshot (sales/cogs/etc.) via getOpenDayPreview.
- Insert into day_closures with the snapshot.
- All of the above in a single batchRun.
- After success: logAudit('إقفال_يومي', detail with totals + variance, 'day_closure', targetDate).

D2.3 — isDayClosed(date) → boolean
  SELECT 1 FROM day_closures WHERE closure_date = ? LIMIT 1

D2.4 — reopenDay(date) → deletes the day_closures row. ADMIN-ONLY (the
caller must wrap in requireAdminAction). logAudit('فتح_يوم_مقفل', date,
'day_closure', date).
